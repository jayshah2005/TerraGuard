# API Documentation

The backend exposes a REST API via FastAPI. The documentation is automatically generated by Swagger UI. Once the backend is running, you can view the interactive documentation at `http://localhost:8000/docs`.

## Base URL
`http://localhost:8000/api/v1`

## Endpoints

### 1. Health Check
*   **GET** `/health`
*   **Description**: Checks if the API is running.
*   **Response**:
    ```json
    {
      "status": "online",
      "version": "1.0.0"
    }
    ```

### 2. Get Region Insights
*   **POST** `/analysis/region`
*   **Description**: Analyzes a region based on coordinates, fetches weather, queries the ML model, and generates IBM watsonx.ai insights.
*   **Request Body** (JSON):
    ```json
    {
      "lat": -13.5320,
      "lon": -71.9675,
      "region_id": "cusco_valley_1"
    }
    ```
*   **Response** (JSON):
    ```json
    {
      "coordinates": {"lat": -13.5320, "lon": -71.9675},
      "environmental_data": {
        "ndvi_current": 0.42,
        "rainfall_30d_mm": 15.2,
        "temp_avg_c": 31.5
      },
      "risk_analysis": {
        "score": 85,
        "level": "High",
        "primary_driver": "Low Rainfall"
      },
      "ai_insight": "Vegetation health is declining rapidly due to sustained high temperatures and lack of rainfall over the past 30 days. High risk of drought stress."
    }
    ```

### 3. Get Historical Weather
*   **GET** `/weather/historical`
*   **Description**: Fetches 30-day historical weather for charting.
*   **Query Parameters**:
    *   `lat` (float): Latitude
    *   `lon` (float): Longitude
*   **Response**: Array of daily weather objects.
